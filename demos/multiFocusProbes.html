Description

This demo allows to interact with node-link diagrams in VR (access for instance on a head-mounted display (HMD) or via an immersive web emulator on a desktop browser). It represents the implementation of the preprint (arXiv), which is to be published as a short paper at the IEEE VIS conference 2025. It uses the web-based 3D engine babylon.js.

Instructions

The demo runs as web-application with the option to enter the VR mode. When the demo-page is accessed on an VR-applicable device like a head mounted display a small icon (see inset) allows to toggle the VR mode. It will load a soccer player data set where all nodes are initially placed w.r.t. to pre-calculated positions from a force directed layout. The following list describes the functionality w.r.t. a Meta Quest 3 (HMD) and further details can be found in the preprint:

  • Button X (left controller) allows to generate a new probe and to place or reselect it. When the placed probe has non-empty intersection with nodes from the node-link diagram, the respective induced subgraph will appear in front of the user.
  • The Thumbstick (left controller) allows to rescale (axes left and right) and translate (axes backward and forward) a probe along the left controller's ray. If a probe is active (see next point) it allows to rotate its content (when the ray hits the sphere surrounding the content). When it is pressed a probe is placed automatically at a node with the largest number of incident links (one representative if not unique).
  • When the left controller ray hits the probe content (the copy in front of the user) then button Y sets the respective probe active, which is visually highlighted by a content-surrounding sphere and a tunnel connecting the probe with its content.
  • Pressing the Squeeze button (left controller) while the left ray hits a node or link (from the whole diagram or the probe content) a slider shows up allowing to change the color of the respective node or link. Pressing the same button lets the slider dissapear.
  • Pressing and holding the Trigger button (left or right controller) allows to replace a probe content in front of the user as long as the respective probe is active.
  • Button A (right controller) allows to remove nodes, links, or probes when the right ray hits the respective object.
  • Button B (right controller) allows to select two nodes (when the right ray hits the respective nodes) in either combination of the whole diagram or probe content and inserts a new link.
  • The Thumbstick (right controller) allows to move (axes backward and forward) in gaze direction whenever the right ray does not hit the sphere surrounding the content of an active probe. If it hits such a sphere it will perform a deformation of the diagram w.r.t. all active probes. Pressing it, changes the layout of the diagram alternating between a circular and a force directed layout, where the latter uses pre-calculated positions.
  • The Squeeze button (right controller) toggles the highlight of the link created at last.
