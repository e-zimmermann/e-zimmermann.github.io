Description

This demo allows to generate geometric shapes in 3D via the evaluation of predefined or user-given functions f, g : ℝ 3 → ℝ 3 throughout three iterative schemes. It represents the implementation of the article published at the Bridges conference 2025 and it uses the web-based 3D engine babylon.js together with GPU-based raymarching. The code is available on github.

Instructions

The demo runs as web-application with the option to enter the VR mode (described below). After launch the settings can be toggled using y on the keyboard and they look like:

The following list describes the settings and expressions/references used therein refer to the article:

  • mult allows to choose among the three implemented iterative schemes: f x g, f x g + c, and f x g + const refer to Equations (3), (4), and (5), respectively.
  • f, g allows to choose among predefined functions and the option user defined. When the latter is chosen, two input fields appear in which functions can be provided in OpenGL Shading Language (glsl). The following expressions are available:
    • p which is a vec3 and gets passed as argument to the multiplication with coordinates p.x, p.y, p.z;
    • r is the Euclidean length of p;
    • cosPhi, sinPhi, cosTheta1, cosTheta2, cosTheta, sinTheta represent the trigonometric evaluations of the angular arguments obtained from p derived from the reformulation of trigonometric functions.
    • The functions provided in the input fields are evlatued after clicking Apply.
  • #iterations determines the number of times the iteration scheme gets evaluated. It ranges between 0 and 100.
  • eps determines the step size for the constant raymarching. It ranges between 1.e-5 and 1.e-2.
  • radius determines the radius of the ball at the origin which is only considered for the raymarching. The square of this value is also used to determine the divergence throughout the iterative process, i.e., when a point throughout the iterative process has length larger than the squared radius the originally considered point in space does not belong to the shape. It ranges between 0 to 10.
  • Julia constant determines the constant used in the iterative scheme from Equation (5). The input field allows to take three float values separated by white space, gets send to the GPU as a vec3, and is evaluated directly.

Rotation and translation in the scene can be achieved by holding the left mouse or right mouse, respectively. The mouse wheel allows to zoom in and out.

VR Mode
When the demo-page is accessed on an VR-applicable device like a head mounted display a small icon (see inset) allows to toggle the VR mode. The following list of controls refer to a Meta Quest 3 headset:

  • The left controller button Y toggles the settings display;
  • The left controller thumbstick allows forward and backward movement in gaze direction.

Gallery

Examples 1 to 3
The following renderings are obtained from the iterative scheme mult = f x g, #iterations = 100, and eps = 5.e-5. The leftmost uses functions f, g = f_A, g_A and a radius = 10. Note that the shape scales w.r.t. the radius without changing its appearance. The second rendering with f, g = f_B, g_B and radius = 6 shows an extension into 3-space suggesting a partition of the same into two components. The rightmost rendering via f, g = f_C, g_C and radius = 10 also extends through space, yet thinning out away from the origin.

Examples 4 and 5 - Spherical Representation
In these examples we can see the spherical representation as an extension of the 2D Mandelbrot set into 3D proposed by Daniel White in 2007. This extension provided the initial motivation to rephrase the multiplication and apply it throughout different schemes. The renderings are obtained using mult = f x g + c, #iterations = 10, radius = 2, and eps = 5.e-5. The functions f, g = f_A, g_A and f, g = f_B, g_B then yield the two renderings on the left and right, respectively.

Example 6 - Opuntia
The last example is related to Julia sets, because now we observe the behavior of the spherical representation for different constants similar to Julia sets in the complex plane. Therefore, we could consider such objects as Julia sets in 3D caused by the spherical representation mentioned above. The following four renderings are obtained using functions f, g = f_A, g_A in the iterative scheme mult = f x g + const with #iterations = 10, eps = 5.e-5, radius = 2, and Julia constant = -1. 0. 0.. The view directions for the second, third, and fourth are along the x-, y-, and z-axis, respectively. We might call this specific object opuntia named after the plant from the cactus family as both share similarites w.r.t. their shapes. We thank Prof. Jörg Arndt (TH Nürnberg, Germany) who suggested that name at the Bridges conference 2025.
